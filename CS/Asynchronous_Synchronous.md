## 동기 VS 비동기

---

### 동기(Synchronous)

- 동기는 요청한 작업이 끝날때까 기다린 후 다음 작업을 수행하는 방식

#### 장점

- 디버깅이 쉽다.
- 실행순서가 명확하여 이해하기 쉽다.

#### 단점

- 다음 작업까지 대기시간이 생긴다.
- 하나의 실행 흐름에서는 대기 시간이 발생한다.
- I/O 작업 (DB, 네트워크)에 취약하다

### 비동기(Asynchronous)

- 비동기는 작업을 요청한 후 기다리지 않고, 다음 작업을 수행하는 방식
- 작업이 완료되면 콜백, 이벤트 등으로 결과를 전달받는다.

#### 장점

- 고성능 서버에 유리하다.
- 동시에 작업 처리 가능하다.
- 호출한 쪽이 대기하지 않는다.

### 면접 예상 질문

1. 동기와 비동기 차이점을 말씀해주세요.
    - 답변: 동기는 작업을 요청한 후 그 결과를 한 흐름에서 바로 처리하는 방식이고,
           비동기는 작업을 요청한 후 다른작업을 하다가 콜백이나 이벤트로 결과를 처리하는 방식입니다.
2. 동기는 어떤경우 비동기는 어떤경우에 사용하면 될까요?
    - 답변: 동기는 실행 순서가 중요한 경우 적합하고, 비동기는 DB조회나 네트워크 통신처럼 I/O 대기시간이 긴 작업이 많고,
           동시에 많은 요청을 처리할 때 적합합니다.
3. 왜 비동기가 자원이 효울적인가요?
    - 답변: 비동기는 I/O작업을 요청한 후 대기하지 않고 다른 작업을 수행할 수 있기 때문에 스레드가 불필요하게 대기 상태에서 머무르지 않습니다.
           그 결과 동일한 자원으로 더 많은 요청을 처리할 수 있어 자원 효율적 입니다.

# CS 주제 - 동기 vs 비동기

## 동기 (Synchronous)

* 요청한 작업이 완료될 때까지 호출자가 결과를 기다린 후 다음 작업을 수행하는 방식이다. \ 작업의 완료 시점과 제어 흐름이 강하게 결합되어 있다.

## 비동기 (Asynchronous)

* 요청한 후 결과를 기다리지 않고 제어 흐름을 반환받아 다른 작업을 수행하다가, 작업 완료 시점에 결과를 처리하는 방식이다.

### 면접 질문 2개

1. 동기와 비동기의 차이점이 무엇인가요? \
    답변: 동기는 요청한 작업의 결과를 받을 때까지 대기한 후 다음 로직을 수행하는 방식이고, \
         비동기는 결과를 기다리지 않고 제어 흐름을 반환받아 다른 작업을 수행하는 방식입니다. \
         핵심 차이는 결과를 기다리느냐와 제어 흐름이 언제 반환되느냐입니다.

2. 응답을 자유롭게 사용하고 여러 작업이 가능한 비동기가 있는데, 동기를 사용하는 이유가 무엇인가요? \
    답변: 동기 방식은 흐름이 직관적이고 구현이 단순하여 \
         결과의 순서 보장이 중요하거나, \
         트랜잭션처럼 작업의 일관성이 중요한 경우에 유리합니다. \
         또한 비동기는 복잡도가 높아 디버깅과 예외 처리가 어렵기 때문에, \
         모든 상황에서 비동기가 적합한 것은 아닙니다.

### 꼬리 질문 2개

1. 동기와 블로킹은 같은 개념인가요?
    답변: 동기/비동기는 결과를 기다리는 방식에 대한 개념이고, 블리콩/논블로킹은 호출 시 제어권을 반환하느냐에 대한 개념입니다. 서로 조합이 가능합니다.
2. Spring에서 비동기 처리는 어떻게 하나요?
    답변: `@Asnyc`를 사용하거나, `CompletableFuture`, `WebClient` 등을 통해 비동기 처리를 구현할 수 있습니다. \
         이 경우 요청 스레드를 즉시 반환하여 자원을 효율적으로 사용할 수 있습니다.

### 왜 중요하다고 생각했는지

웹 서버에서는 대부분의 요청이 DB 조회나 외부 API 호출과 같은 I/O 작업이기 때문에,
동기와 비동기 처리 방식에 따라 성능과 확장성에 큰 차이가 발생한다.
또한 이 개념은 서버 스레드 모델과 장애 전파 방식에도 직접적인 영향을 주기 때문에
백엔드 개발자에게 필수적인 개념이라고 생각했다.
